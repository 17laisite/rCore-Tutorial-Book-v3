移除标准库依赖¶

让我们从零开始创建内核项目。一开始，它就和一个普通的 Cargo 项目没有什么不同：

$ cargo new os --bin

我们加上了 --bin 选项来告诉 Cargo 我们创建一个可执行项目而不是库项目。此时，项目的文件结构如下：

$ tree os
os
├── Cargo.toml
└── src
    └── main.rs

1 directory, 2 files

其中 Cargo.toml 中保存着项目的配置，包括作者的信息、联系方式以及库依赖等等。显而易见源代码保存在 src 目录下，目前为止只有 main.rs 一个文件，让我们看一下里面的内容：
